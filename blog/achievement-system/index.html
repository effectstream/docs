An Open Achievement Standard for On-Chain Games

April 28, 2026 · 5 min read
EffectStream

Steam has achievements. Xbox has Gamerscore. PlayStation has Trophies. But on-chain games have no standard way to track player accomplishments across games. We built one — an open standard that any EffectStream game can implement, plus a portal where players can see all their achievements in one place.

Defining the Standard​

Every platform-locked achievement system (Steam, Xbox, PlayStation) shares the same limitation: your gaming history is tied to your account on that platform. If you switch, you start from zero. On-chain games have the opportunity to do better — achievements tied to a wallet are portable by default.

We created PRC-1 (Paima Request for Comment), a community standard similar to CIPs for Cardano or ERCs for Ethereum. PRC-1 defines how games register, track, and expose achievements via a standard API. Any EffectStream node that implements PRC-1 exposes two HTTP endpoints:

  • GET /achievements/public/list — returns all possible achievements in a game, their descriptions, and global completion statistics
  • GET /achievements/wallet/:walletAddress — returns a specific player's progress, including which achievements they've unlocked, completion dates, and progress toward incomplete ones

The standard was later extended into PRC-6, which adds metric channels and leaderboard rankings for integration with the broader Midnight Platform. PRC-6 defines a richer set of endpoints (/metrics, /metrics/{channel}, /metrics/users/{address}) that allow the platform to aggregate data across all participating games.

Proof of Concept — Integrating Multiple Games​

To validate the standard, we integrated it into multiple live games. The achievement system is now running in:

  • Dust-2-Dust — ZK deck-building dungeon crawler
  • Block Kart Legends — cross-chain EVM + ZK racing
  • Safe Solver — L2 risk-reward game on Midnight
  • Kachina Kolosseum — ZK commit-reveal PvP arena
  • Tower Defense — open-source tower defense template

Integrating achievements into a game is straightforward. You define your achievements as metadata in your API entry point, and EffectStream handles the rest — exposing PRC-1 compliant endpoints automatically:

const achievementList: Achievement[] = [
    {
        name: "finish-chapter-1",
        displayName: "Over The River",
        description: "Finish Chapter 1.",
        isActive: true,
    },
    {
        name: "win-10-battles",
        displayName: "Battle Hardened",
        description: "Win 10 battles against any opponent.",
        isActive: true,
    },
];

const metadata: AchievementMetadata = {
    game: { id: "my-awesome-game" },
    list: achievementList,
};

export const achievements = Promise.resolve(metadata);

Tracking progress happens inside your State Transition Functions. When a player performs an action that matters for an achievement, you read their current progress and update it:

import { getAchievementProgress, setAchievementProgress } from '@effectstream/db';

async function handleBattleWin(walletId: number, blockTimestamp: Date, dbConn: Pool) {
  const current = await getAchievementProgress.run(
    { wallet: walletId, names: ['win-10-battles'] }, dbConn
  );
  const row = current[0];
  if (!row?.completed_date) {
    const newProgress = (row?.progress ?? 0) + 1;
    return [[setAchievementProgress, {
      name: 'win-10-battles',
      wallet: walletId,
      completed_date: newProgress >= 10 ? blockTimestamp : null,
      progress: newProgress,
      total: 10,
    }]];
  }
  return [];
}

You can see a real-world example in the Tower Defense backend, where achievements are registered in a few lines at the game's entry point.

Designing the Achievement Portal​

With every game exposing the same API, we could build a portal that aggregates achievements from all of them. Players connect their wallet and see all their accomplishments in one place — across every EffectStream game they've played.

The portal doesn't use a centralized database. It queries each game's PRC-1 endpoints at runtime and presents a unified view. This means any new game that implements the standard automatically appears in the portal — no integration work required on the portal side.

We designed the portal UI in Figma, focusing on a clean presentation that surfaces progress across games, completion percentages, and unlocked achievements with timestamps.

The portal connects to the PRC-1 endpoints of each registered game and aggregates the data into a single player profile. When you connect your wallet, you see every achievement you've earned across Dust-2-Dust, Block Kart Legends, Safe Solver, Kachina Kolosseum, and any future game that implements the standard.

The Achievement Ecosystem​

The achievement system is now live across 5+ games, with the achievement code built directly into EffectStream. Any new game built with the framework gets achievement support out of the box — define your achievements, track progress in your STFs, and the API is served automatically.

Unlike platform-locked achievements, these are on-chain and portable. Your gaming history follows your wallet, not your platform account. And because the standard is open, anyone can build their own portal, integrate achievement data into their application, or create cross-game experiences that reward players for their accomplishments across the entire ecosystem.
