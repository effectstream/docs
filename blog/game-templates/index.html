Game Template Library: Five On-Chain Game Patterns

April 28, 2026 · 8 min read
EffectStream

The best way to learn a framework is to see it in action. We built five distinct game templates — each showcasing a different chain integration, cryptographic technique, or architectural pattern. All are open-source and playable live. This article walks through each one and explains the technology behind it.

Trading Card Game — Cryptographic Card Mechanics​

How do you deal cards fairly on a public blockchain? Every transaction is visible, so a naive implementation would let opponents see your hand. We solved this with a novel cryptographic protocol for hidden-hand card games.

The core problem is that there's no trusted dealer in a decentralized game. Both players need to agree on a shuffled deck without either side being able to peek. Our approach uses a commit-reveal cryptographic scheme where the deck state is committed as hashes, and card reveals are verified against those commitments. The protocol guarantees that neither player can manipulate the shuffle or see cards they shouldn't — the same fairness properties as a physical deck, but enforced by math instead of trust.

The template implements a full trading card game with deck building, hand management, and turn-based combat — all running on-chain through EffectStream's state machine.

Safe Solver — EffectStream as an L2 on Midnight​

EffectStream provides a TypeScript L2 runtime that merges blocks from multiple chains — EVM and Midnight ZK — into a single unified stream for executing game logic. Safe Solver is the template that demonstrates this architecture in a playable game.

The game itself is a risk-reward safe-cracking challenge built with Three.js. Each level presents 3–7 safes, with only one hiding an alarm. Players drill safes to earn gold, with escalating risk: fewer safes mean higher base points but worse odds. Bank your points with "KEEP REWARD" or push your luck — trigger the alarm and lose everything unbanked. Scores are saved to a leaderboard on the Midnight blockchain.

What makes this architecturally interesting is how EffectStream acts as the execution layer. The game's TypeScript state machine processes turns, computes outcomes, and manages player state, while Midnight provides the privacy-preserving settlement layer and EVM handles token contracts. EffectStream merges both chains' blocks into a single stream, so the game logic sees one coherent timeline regardless of which chain originated the event.

Kachina Kolosseum — ZK Commit-Reveal PvP​

In a PvP game on a public blockchain, the second player has an unfair advantage: they can see the first player's move on-chain and counter it. ZK commit-reveal eliminates this problem.

Kachina Kolosseum is a turn-based arena battler where two players face off with teams of 3 heroes. Each hero has equipment loadouts (axes, shields, bows, swords, spears) and stances (Defensive, Neutral, Aggressive) that multiply damage. The combat flow works like this:

  1. Player 1 commits — selects attack targets and stances, then submits a zero-knowledge hash of those choices. The moves are hidden on-chain.
  2. Player 2 submits — moves openly, since Player 1's commitment is already locked.
  3. Player 1 reveals — the ZK circuit verifies the revealed moves match the original hash, then resolves damage for both sides simultaneously.

This prevents front-running or move-snooping. If Player 1 tries to reveal different moves than they committed, the ZK proof fails and the transaction is rejected. If a player refuses to reveal, the game can resolve with a timeout penalty.

Block Kart Legends — Cross-Chain EVM + ZK Racing​

Block Kart Legends demonstrates how EffectStream's L2 can combine EVM and Midnight in parallel to produce complex game outcomes. The engine merges results from both chains into a virtual execution layer where heavyweight computations — like simulating an entire race with thousands of steps and variables — can take place in TypeScript while both chains provide the trust guarantees.

Players allocate 25 stat points across Speed, Acceleration, Weight, Handling, and Traction, then select items for strategic use during the race. The backend simulates the full race using on-chain data, while the Three.js frontend runs a parallel simulation for immediate visual feedback. The ZK layer on Midnight ensures race configuration integrity, and the EVM layer on Arbitrum handles the token and reward contracts.

This template proves that EffectStream's multi-chain merge architecture isn't just for turn-based games. Real-time simulations with complex physics and thousands of state transitions can run on the L2, anchored to both a privacy-preserving ZK chain and a high-throughput EVM chain.

Dust-2-Dust — Pushing the Limits of ZK Contracts​

Dust-2-Dust is a fully on-chain deck-building dungeon crawler where all game logic is secured by zero-knowledge proofs on the Midnight network. It's our most technically ambitious template — not because of the game design, but because of what we had to solve to make ZK contracts handle this level of complexity.

ZK circuits are fundamentally constrained. Even simple operations are expensive to prove, and Midnight's Compact language compiles down to circuits where every branch and lookup costs real proof-generation time. A combat round in Dust-2-Dust involves: resolving 3 player abilities (each with primary effects, energy-triggered secondary effects, and elemental types), computing damage against up to 3 enemies (each with physical/fire/ice resistances, blocking, healing, and ally buffs), rotating a 7-card deck with non-repeating draws, and updating all state — in a single ZK circuit.

We developed novel techniques to make this work efficiently in Compact:

  • Witness-validated division: ZK circuits don't natively support division, so we use an external witness that provides the quotient and remainder, then validate x == y * quotient + remainder inside the circuit.
  • Boolean arithmetic for conditional math: to avoid underflow checks in non-executed branches, all conditional damage calculations use multiplication by boolean flags ((condition as Uint<1>) * value) instead of if/else.
  • Dynamic indexing via circuit unrolling: ZK circuits only support constant indices, but deck rotation requires dynamic access. We unroll lookups into exhaustive if-chains over all possible indices.
  • Deterministic per-player RNG: each player has a persistent hash chain (persistentHash of the previous seed), ensuring reproducible randomness without a trusted oracle.

The game features biome-based quest progression, boss battles with increasing difficulty gates, an ability upgrade system (sacrifice + gold cost), and an energy-color synergy system where abilities trigger bonus effects based on teammates' energy generation. The entire 970-line Compact contract runs in ZK — every combat round, every ability upgrade, every quest finalization produces a zero-knowledge proof.

Five templates, five different approaches to on-chain gaming:

  • Trading Cards — cryptographic card mechanics without a trusted dealer
  • Safe Solver — EffectStream as an L2 merging EVM + Midnight into one stream
  • Kachina Kolosseum — ZK commit-reveal for fair PvP combat
  • Block Kart Legends — cross-chain EVM + ZK racing with complex simulation
  • Dust-2-Dust — a full dungeon crawler pushing ZK contract complexity to its limits

These aren't just demos. Each template is a complete, deployable game that developers can fork, modify, and ship. The source code is open and the live versions are playable today.
