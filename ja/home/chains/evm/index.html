EVM-Compatible Chains

EVM (Ethereum Virtual Machine) chains are common and widely supported smart contract platforms in the Web3 ecosystem. This includes Ethereum itself, as well as numerous Layer 2 solutions like Arbitrum, Optimism, and Polygon, and sidechains like Avalanche C-Chain.

For most dApps built with Effectstream, an EVM chain will serve as a primary layer for asset management (NFTs, tokens) and user interactions via the PaimaL2Contract.

Configuration‚Äã

You add an EVM network to your Paima application using the addViemNetwork helper in your config.ts, which leverages the chain definitions from the popular viem library.

  • Network Definition (buildNetworks):

    .buildNetworks(builder =>
      builder.addViemNetwork({
        ...hardhat, // Or ...arbitrum, ...mainnet, etc.
        name: "myEvmChain",
      })
    )

  • Sync Protocol Definition (buildSyncProtocols): The protocol type for connecting to an EVM chain is EVM_RPC_PARALLEL.

    .addParallel(
      (networks) => networks.myEvmChain,
      (network, deployments) => ({
        name: "myEvmRpcProtocol",
        type: ConfigSyncProtocolType.EVM_RPC_PARALLEL,
        chainUri: network.rpcUrls.default.http[0],
        startBlockHeight: 1,
        pollingInterval: 1000,
        confirmationDepth: 2, // Wait for 2 blocks before considering an event final
      }),
    )

Supported Primitives‚Äã

Paima provides a rich set of primitives for automatically tracking common on-chain activities on EVM.

  • EvmRpcPaimaL2: Listens for PaimaGameInteraction events from the PaimaL2Contract.
  • EvmRpcERC20: Tracks Transfer events from an ERC20 contract and automatically maintains a table of token balances.
  • EvmRpcERC721: Tracks Transfer events from an ERC721 contract and automatically maintains a table of NFT owners.
  • EvmRpcERC1155: Tracks TransferSingle and TransferBatch events from an ERC1155 contract.
  • EvmRpcGeneric: A powerful generic primitive that can be configured to listen for any event from any contract, given its ABI.

Contract Development‚Äã

  • Language: Solidity
  • Tools: The Paima templates are pre-configured for development with Hardhat and Foundry (Forge).
  • Compilation: deno task build:evm

Local Development Setup‚Äã

The launchEvm function provided by the Paima Orchestrator (@effectstream/orchestrator/start-evm) automates your local setup. When included in your orchestrator.ts, it will:

  1. Start a local Hardhat node.
  2. Wait for the node to be responsive.
  3. Run your deployment scripts to deploy all your EVM contracts.
